## Технічне завдання 

**на розробку Єдиної інформаційної системи соціальної підтримки громадян (ЄІСС-2025)**

**1. Вступ**

1.1. **Назва проєкту:** Єдина інформаційна система соціальної підтримки громадян (ЄІСС-2025)

1.2. **Зокрема:** Це технічне завдання (ТЗ) визначає технічні вимоги до розробки, впровадження та сертифікації Єдиної інформаційної системи соціальної підтримки громадян (ЄІСС-2025).

1.3. **Мета проєкту:** Автоматизація процесів надання соціальної допомоги громадянам України, підвищення ефективності роботи органів соціального захисту та поліпшення якості обслуговування.

1.4. **Замовник:** Міністерство соціальної політики України

1.5. **Виконавець:** Державне підприємство «Інфотех»

**2. Підстави для розробки ТЗ**

2.1. **Наказ Міністра соціальної політики України:** №55/2025 від 10.02.2025 "Про затвердження проєкту Єдиної інформаційної системи соціальної підтримки громадян".

**3. Загальні характеристики проєкту**

3.1. **Тип проєкту:** Створення з нуля (Greenfield)

3.2. **Термін виконання:** з 01.03.2025 по 31.12.2025

3.3. **Етапи розвитку:**

3.4. **Бюджет:** 50 000 000 грн.

**4. Опис функціональних вимог**

4.1. **Очікувані результати:**

* Автоматизація 90% виплат соціальної допомоги.
* Зменшення часу обробки заяви до 15 хвилин.
* Оптимізація роботи соціальних інспекторів та операторів ЦНАП.
* Збільшення прозорості та доступності інформації для громадян.
* Зниження витрат на паперовий документообіг.

4.2. **Використовувані сценарії:**

* **UC-1: Подання заяви через Дію.** Громадянин подає заяву на отримання соціальної допомоги через мобільний додаток "Дію".

* **UC-2: Автоматична перевірка доходів через ДПС.** Система автоматично перевіряє доходи заявника в базі даних ДПС.

* **UC-3: Формування платіжної відомості.** Система генерує платіжну відомість для виплати соціальної допомоги.

* **UC-4: Генерація звітності для Мінфіну.** Система генерує звіт про використання коштів на соціальну допомогу для Міністерства фінансів України.

**5. Технічні вимоги**

5.1. **Архітектура:**

* **Стиль:** Microservices.

* **Оґрунтування:** Необхідність незалежного масштабування модулів виплат та високе навантаження в пікові періоди.

* **Інфраструктура:**

* **Deployment model:** Private Cloud (G-Cloud)

* **Cloud provider:** De Novo / GigaCloud

* **Containerization:** Kubernetes (K8s)

* **Data architecture:**

* **DB types:** PostgreSQL, MongoDB (для логів), Redis

* **DB pattern:** Database per Service

* **Volume and backup:** Прогноз: 50 ТБ даних/рік.

Повний бекап щодня, інкрементальний кожні 4 години.

* **Integrations:**

* **Patterns:** REST API (зовнішні), gRPC (внутрішні), Kafka (асинхронні події)

* **Details:** API Gateway: Kong. Синхронна обробка для UI, Асинхронна для звітів.

5.2. **Qualіty Assurance:**

* **Critical:** Security, Availability, Data Integrity

* **High:** Scalability, Performance

5.3. **ADR:**

* **ADR-1:** Використання Event Sourcing для історії нарахувань.

* **ADR-2:** Зберігання файлів у S3-сумісному сховищі.

5.4. **Нефункціональні вимоги:**

* **Performance:** API response < 200ms, генерація звіту < 5 хв

* **Scalability:** Підтримка до 100 000 одночасних користувачів

* **Availability:** 99.9% (SLA), downtime не більше 45 хв/міс

* **Security:**

* КСЗІ Г2
* Шифрування GOST/AES
* MFA для адміністраторів
* SIEM-моніторинг

5.5. **Технічний стек:**

* **Frontend:** React, TypeScript, Next.js

* **Backend:** Java (Spring Boot) / Python (FastAPI)

* **Infrastructure:** Terraform, Ansible, ArgoCD, Prometheus/Grafana

* **Software description:** Сучасний стек на базі Open Source рішень.

* **Integrations list:** Діа, ЄДР, ДРАЦС, Реєстр територіальної громади, Трембіта.
